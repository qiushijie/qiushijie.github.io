npm项目本地部署：使用npm link

发表于 2016-10-06   |   分类于 node.js   |  

有的时候，我们开发的私有公共项目，其他项目需要引用，但是又不希望发布到npm仓库中（下一篇文章将介绍怎么搭建私有npm仓库），或者是该私有项目还需要不断的去调试，在其他项目中调用的过程中出现的问题能及时修复。这样的情况下，在本地直接依赖该私有项目是一个比较不错的解决方案。

安装

先进入到需要发布的项目目录，执行sudo npm link命令，该命令将项目link到全局的node_modules目录中

引用

进入到需要引用该私有项目的目录中，执行npm link 私有项目名称即可

编译安装

接下来介绍一下另外一种情况，如果项目中使用了babel等工具对代码进行编译，我们直接在项目的根目录link会导致引用出现异常，因为这里的代码是还没有经过编译的。例如我们编译后的代码放在dist目录中，先将package.json等文件拷贝过去，最好是写成一个build命令，因为在发布的时候也会用到。然后进入dist目录，在dist目录下去执行sudo npm link即可。如果项目添加了新的依赖，需要重新进入到dist目录里面来运行sudo npm install命令。
