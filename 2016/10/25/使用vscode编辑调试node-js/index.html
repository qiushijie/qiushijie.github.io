使用vscode编辑调试node.js

发表于 2016-10-25   |   分类于 编辑器   |  

tasks

在.vscode目录内新建tasks.json文件，tasks可以指定运行特定的命令，可以使用babel自动编译修改后的代码。添加一个运行tasks的快捷键，打开快捷键设置加入

按下快捷键后直接选择运行相应的task

babel

加入编译task，并设定isWatching为true

多task

vscode的tasks默认有一个command命令，执行的tasks是在这个command内的，如果要在task内做多个不同命令的操作，可以设定运行的命令为shell。

运行单个js文件

点击需要运行的文件，cmd + r，选择runFile就可以运行了。

运行单个mocha测试

点击需要运行的测试文件，cmd + r选择runTest运行。

tasks示例

可以运行单个文件，也可以运行单个测试，使用babel编译

debug

在.vscode目录内新建launch.json文件，launch.json文件是vscode用于运行和调试的配置文件。

launch示例

以下配置文件为运行express的配置

参数说明

program

程序入口文件

env

node 运行环境变量

runtimeArgs

node 运行参数，如"runtimeArgs" : ["--harmony"]

sourceMaps

指定是否使用sourceMaps，默认为false

outFiles
