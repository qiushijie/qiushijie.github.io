ES7 async/await异步编程

发表于 2016-10-06   |   分类于 node.js   |  

async await

async和await是ES7新标准提出的解决异步编程中回调嵌套问题的关键词，其实async和await相当于yield和*，async、await语义更加清晰。async和await可以说是作为JavaScript异步编程的解决的终极方案，使用起来非常方便。在编写node.js代码时，不会再出现回调地狱的问题，代码的结构也更为清晰。async关键词写在一个函数声明的前面，其含义是该函数是一个异步的函数。await用在async函数体内，当执行到有await语句的地方，会先执行完该异步语句，然后再往下去执行下一条语句。

node.js 7.0中已经可以直接使用–harmony直接支持async，现在一般通过babel插件来将async函数编译然后执行。现在node.js 7还没有正式发布，所以先介绍一下怎么样使用babel来支持async。

sequelize系列教程4：关系

发表于 2016-10-05   |   分类于 node.js   |  

示例

介绍sequelize关系使用一个rbac用户权限系统来进行，通过该系统来对sequelize的关系使用进行讲解。

一个用户属于一个用户分组

通过设置as属性可以在实例中使用get/set方法访问到关联的模型

一个用户分组拥有多个用户

在sequelize直接设置双向的关系并不能起作用，而且会在单方生成两个外键，所以我们需要通过外键的设置来解决这个问题，让用户可以访问到用户分组，用户分组可以获取到其所有的用户。最终的关系设置应该是这样的

一个用户分组可以有多个角色，一个角色可以有多个用户分组，这是属于多对多的关系，多对多模型在sequelize中用belongsToMany表示，通过一个through的中间模型来管理多对多的双边关系，through可以是一个model，也可以是一个字符串。singular和plural表示as的单数和复数，可以通过singular直接操作一个的项，plural和普通的as一样。

关系详解

belongsTo

belongsTo可以表示一个一对一的属于关系，也可以表示一个多对一的属于关系。例如一个身份证信息属于一个用户，一个用户属于一个用户分组

hasOne

hasone表示一个一对一的拥有关系，例如一个用户拥有一个身份证信息

hasMany

hasMany表示一个一对多的关系的拥有关系

belongsToMany

belongsToMany表示一个多对多的关系，例如一个用户分组拥有多个角色，一个角色拥有多个用户分组。或者是一篇文章拥有多个tag，一个tag拥有多篇文章。
