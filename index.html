文法定义

根据json的语法格式，写出一个json的antlr文法。

json

json由一个object({…})或者array([…])组成，文法是

object

object以一个’{‘符号开始，以一个’}’符号结束，由一个或者多个键值(key : value, …)对组成，文法是

array

array以’[‘符号开始，’]’符号结束，又一个或者多个值(value, …)组成，所以文法定义是

pair

一个键值对由一个字符串作为键，一个值组成，文法定义是

value

一个值可以是字符串，也可以是数字，也可以是一个bool，也可以是null。值还可以是一个object或者array，表示嵌套的json格式。文法定义是

STRING

字符串由’”‘开始，’”‘结束，’\”‘是反义符号，在字符串内表示‘“’

小结

编译原理在编程中有很广泛的应用，在java中，像hibernate等的orm框架使用antlr来解析sql语句，在beelt模板引擎中使用antlr来解析模板文件。在外部dsl、代码生成、静态分析等领域也有很广泛的应用。antlr的文法比较简单，入门门槛也不高，编译原理没有想象中这么困难。

npm项目本地部署：使用npm link

Posted on 2016-10-06   |   In node.js   |  

有的时候，我们开发的私有公共项目，其他项目需要引用，但是又不希望发布到npm仓库中（下一篇文章将介绍怎么搭建私有npm仓库），或者是该私有项目还需要不断的去调试，在其他项目中调用的过程中出现的问题能及时修复。这样的情况下，在本地直接依赖该私有项目是一个比较不错的解决方案。

安装

先进入到需要发布的项目目录，执行sudo npm link命令，该命令将项目link到全局的node_modules目录中

引用

进入到需要引用该私有项目的目录中，执行npm link 私有项目名称即可

编译安装

接下来介绍一下另外一种情况，如果项目中使用了babel等工具对代码进行编译，我们直接在项目的根目录link会导致引用出现异常，因为这里的代码是还没有经过编译的。例如我们编译后的代码放在dist目录中，先将package.json等文件拷贝过去，最好是写成一个build命令，因为在发布的时候也会用到。然后进入dist目录，在dist目录下去执行sudo npm link即可。如果项目添加了新的依赖，需要重新进入到dist目录里面来运行sudo npm install命令。

ES7 async/await异步编程

Posted on 2016-10-06   |   In node.js   |  

async await

async和await是ES7新标准提出的解决异步编程中回调嵌套问题的关键词，其实async和await相当于yield和*，async、await语义更加清晰。async和await可以说是作为JavaScript异步编程的解决的终极方案，使用起来非常方便。在编写node.js代码时，不会再出现回调地狱的问题，代码的结构也更为清晰。async关键词写在一个函数声明的前面，其含义是该函数是一个异步的函数。await用在async函数体内，当执行到有await语句的地方，会先执行完该异步语句，然后再往下去执行下一条语句。

node.js 7.0中已经可以直接使用–harmony直接支持async，现在一般通过babel插件来将async函数编译然后执行。现在node.js 7还没有正式发布，所以先介绍一下怎么样使用babel来支持async。

sequelize系列教程4：关系

Posted on 2016-10-05   |   In node.js   |  

示例

介绍sequelize关系使用一个rbac用户权限系统来进行，通过该系统来对sequelize的关系使用进行讲解。

一个用户属于一个用户分组

通过设置as属性可以在实例中使用get/set方法访问到关联的模型

一个用户分组拥有多个用户

在sequelize直接设置双向的关系并不能起作用，而且会在单方生成两个外键，所以我们需要通过外键的设置来解决这个问题，让用户可以访问到用户分组，用户分组可以获取到其所有的用户。最终的关系设置应该是这样的

一个用户分组可以有多个角色，一个角色可以有多个用户分组，这是属于多对多的关系，多对多模型在sequelize中用belongsToMany表示，通过一个through的中间模型来管理多对多的双边关系，through可以是一个model，也可以是一个字符串。singular和plural表示as的单数和复数，可以通过singular直接操作一个的项，plural和普通的as一样。

关系详解

belongsTo

belongsTo可以表示一个一对一的属于关系，也可以表示一个多对一的属于关系。例如一个身份证信息属于一个用户，一个用户属于一个用户分组

hasOne

hasone表示一个一对一的拥有关系，例如一个用户拥有一个身份证信息

hasMany

hasMany表示一个一对多的关系的拥有关系

belongsToMany

belongsToMany表示一个多对多的关系，例如一个用户分组拥有多个角色，一个角色拥有多个用户分组。或者是一篇文章拥有多个tag，一个tag拥有多篇文章。
